Template engine

Django’s default template engine is a powerful tool for rendering HTML with dynamic data. It allows you to define templates, which are HTML files with placeholders and logic for inserting data. Here’s an overview of the key features and how you can use them with examples.

1. Template Syntax

Django templates use a special syntax to insert dynamic content and logic. This syntax includes:

  • Variables: Inserted with double curly braces { { } }.
  • Tags: Provide logic like loops and conditionals, enclosed in { % % }.
<h1>Welcome, { { user.name } }!</h1> { % if user.is_authenticated % } <p>You are logged in.</p> { % else % } <p>You are not logged in.</p> { % endif % } <h2>Items:</h2> <ul> { % for item in items % } <li>{ { item.name } } - { { item.price } }</li> { % endfor % } </ul>

2. Template Inheritance

Django supports template inheritance, allowing you to define a base template that other templates can extend.

<head> <title>{ % block title % }My Site{ % endblock % }</title> </head> <body> <header> <h1>My Site</h1> <nav> <ul> <li><a href="/">Home</a></li> <li><a href="/about/">About</a></li> </ul> </nav> </header> <main> { % block content % }{ % endblock % } </main> </body>

3. Template Filters

Django provides built-in filters for transforming variables within templates.

<p>Today is {{ date | date:"F d, Y" }}</p> <p>{{ description | truncatewords:30 }}</p>

4. Template Tags

Template tags are used to perform more complex logic within templates.

  • { % for % }: Loop over items
  • { % if % }: Conditional statements
  • { % include % }: Include another template
